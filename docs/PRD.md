# 產品需求文件 (PRD) - 線上算命系統

## 1. 專案概述
- **背景與動機**：現代人生活節奏快，壓力大，在面臨人生抉擇、感情或事業困惑時常常需要心靈寄託與指引。傳統的實體算命或廟宇抽籤受到時間與空間的限制，需親自到訪。建立一個隨時隨地都能進行線上算命、占卜與抽籤的系統，能輕易滿足大眾透過行動裝置或電腦獲取心靈解答的需求。
- **目標用戶**：各年齡層有心理諮詢、求神解惑或對命理有興趣的大眾。
- **核心價值主張**：提供便捷、隱私且即時的線上算命與占卜服務，讓用戶能隨時獲得指引，並且保留線上添香油錢等擬真體驗，完善線上參拜的儀式感。

## 2. 功能需求
依據核心方向，系統主要提供以下功能：

### 2.1 算命、占卜、抽籤
- 作為 **使用者**，我希望 **能在網站上進行算命、占卜或抽籤等操作**，以便 **獲得對現階段問題或未來的指引解答**。

### 2.2 占卜詳解
- 作為 **使用者**，我希望 **在獲得占卜與抽籤結果後，能查閱詳細的解籤與分析**，以便 **清楚理解結果背後的意涵與具體建議**。

### 2.3 會員註冊與登入
- 作為 **使用者**，我希望 **能通過 Email 或帳號密碼註冊並登入專屬的會員帳號**，以便 **享有更完整的個人化服務**。

### 2.4 儲存算命結果
- 作為 **已登入的會員**，我希望 **系統能自動儲存我每一次的算命或抽籤紀錄**，以便 **我日後隨時登入查看、回顧過去的測算結果**。

### 2.5 捐香油錢
- 作為 **使用者**，我希望 **能透過線上介面選擇捐獻香油錢**，以便 **為自己或親友祈福，並將心意回饋給系統或神明**。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：使用 Flask。
  - 前端渲染：使用 Jinja2 模板，搭配 HTML/CSS/JS 開發，確保有優良的響應式設計 (RWD)。
  - 資料庫：使用 SQLite 儲存會員資料、測算結果與香油錢紀錄。
- **效能與安全考量**：
  - 密碼必須進行安全雜湊加密（如 bcrypt）儲存。
  - 需要考慮金流行為的安全性，若為模擬金流須明確提示，若串接真實第三方金流須妥善保護連線資訊與 Token。
  - 防止 SQL Injection 與 XSS 攻擊。

## 4. MVP 範圍

### Must Have (必須擁有)
- 會員的基礎註冊、登入與登出系統
- 至少實作一種核心測算模式（例如：線上廟宇抽籤）
- 籤詩或結果的詳解展示頁面
- 個人中心頁面，展示會員過去算命/抽籤的歷史紀錄

### Should Have (應該擁有)
- 線上添香油錢的基礎功能（可以先開發表單與模擬付款流程）
- 更多元的測算模式（例如：星座占卜、塔羅牌抽牌等）

### Nice to Have (可以擁有)
- 串接真實金流服務（Line Pay, 綠界等）進行捐款
- 結果分享功能，可將算命詳解一鍵分享至社群平台 (FB, LINE)
